# sfsync.sh

Salesforce組織の変更を検出し、Gitリポジトリへ自動同期するためのシェルスクリプト。

## 1. 概要

Salesforce（ターゲット組織: `tama`）で更新されたメタデータを抽出し、ローカルのGitブランチ（`development`等）へ反映、コミット、プッシュまでを一気通貫で行います。

## 2. 設計ポリシー

* **不純物の排除**: 空白行にはインデントを一切含まず、完全な空行（0文字）を維持しています。
* **1行1命令**: セミコロンによる命令の連結を禁止し、垂直方向の論理構造を明確化しています。
* **高密度ロギング**: 成功時はノイズとなる詳細出力を抑制し、失敗時のみ調査に必要な生データを記録します。
* **完全な透明性**: 実行されるすべてのコマンドをログに刻印し、裏側での隠れた動作を排除しています。

## 3. 同期プロセス

スクリプトは以下の5つのフェーズで構成されます。

| フェーズ | ステージ | 処理内容 |
| --- | --- | --- |
| **Step 1** | **GIT** | `stash` で環境を保護し、`pull --rebase` で最新状態を取得。 |
| **Step 2** | **DELTA** | `sf sgd` を使用し、前回同期時からの変更箇所を特定。 |
| **Step 3** | **RETRIEVE** | 変更箇所および主要メタデータ（Apex, Layout等）を取得。 |
| **Step 4** | **CLEAN** | 作業用の一時ディレクトリ（`temp_delta`）を完全に削除。 |
| **Step 5** | **SYNC** | 差分がある場合のみ、コミットおよびプッシュを実行。 |

## 4. ログ出力仕様

ログファイル `sfsync_latest.log` は実行のたびに上書き更新されます。

* **[INFO]**: フェーズの開始および進行状況の説明。
* **[CMD]**: 実際に実行されたコマンドの全文。
* **[SUCCESS]**: 各フェーズが正常に完了したことを示す。
* **[ERROR]**: 異常終了時の記録。失敗時のみコマンドの出力詳細（JSON等）を追記。

## 5. 終了コード

| コード | 状態 | 詳細 |
| --- | --- | --- |
| **0** | **SUCCESS** | すべての工程が正常に完了。 |
| **1** | **ERROR** | 工程の途中でエラーが発生し、処理を中断。 |
| **2** | **NO CHANGE** | Salesforce組織側に変更がなく、同期をスキップ。 |
