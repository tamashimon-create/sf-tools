# sf-tools

Salesforce開発における、面倒な環境構築や日々の作業を自動化するためのシェルスクリプト集です。
`sf-start.sh` コマンド一つで、開発の準備がすべて整います。

## 1. 概要

*   **ワンコマンド・スタートアップ**: `sf-start.sh` を実行するだけで、環境チェック、Salesforce組織への接続、VS Codeの起動までを自動で行います。
*   **環境の自動セットアップと統一**: チームメンバーのローカル環境に必要なツールや設定を自動で同期し、環境差異による問題を未然に防ぎます。
*   **Gitフックによる品質担保**: `git push` する前に、Salesforce組織へのデプロイ検証を自動実行する `pre-push` フックを簡単に導入できます。
*   **柔軟なリリース管理**: ブランチごとに用意されたテキストファイル (`deploy-target.txt`) に対象コンポーネントのパスを記述するだけで、`sf-release.sh` がマニフェストを自動生成し、安全なデプロイ・検証を実行します。

## 2. 使い方

### 初回セットアップ

1.  **リポジトリのクローン**:
    `sf-tools` をホームディレクトリなど、任意の場所にクローンします。
    ```bash
    git clone <sf-toolsのリポジトリURL> ~/sf-tools
    ```

2.  **プロジェクトへの導入**:
    開発したいSalesforceプロジェクトのルートディレクトリに、`sf-tools` をインストールするためのスクリプト (`sf-install.sh`) を配置し、実行します。
    （`sf-install.sh` は、`sf-tools` の設定をプロジェクトに適用する役割を担います）

### 日常的な開発フロー

1.  Salesforceプロジェクトのルートディレクトリで、以下のコマンドを実行します。
    ```bash
    bash sf-start.sh
    ```
    ※ `sf-install.sh` によって、この `sf-start.sh` はプロジェクト内に自動で配置されます。

2.  スクリプトが自動で環境チェック、Salesforce組織への接続確認を行います。

3.  接続されていない場合は、組織のエイリアスを入力し、ブラウザでログイン認証を行います。

4.  すべての準備が完了すると、自動でVS Codeが起動します。

## 3. 提供されるスクリプト

| スクリプト | 概要 |
|:---|:---|
| `sf-start.sh` | **メインスクリプト。** 開発環境のセットアップからVS Code起動までを一括で行います。 |
| `sf-release.sh` | **リリース/検証スクリプト。** `deploy-target.txt` 等からマニフェストを生成し、Salesforce組織へデプロイまたは検証を実行します。 |
| `sf-inithooks.sh` | Gitの `pre-push` フックを現在のプロジェクトにインストールします。 |
| `sf-unhook.sh` | `pre-push` フックを無効化（削除）します。 |
| `hooks/pre-push` | `git push` 時に `sf-release.sh` を呼び出し、デプロイ検証を自動実行するフック本体です。 |

## 4. ワークフロー詳細

### `sf-start.sh` の処理フロー

| ステップ | 処理内容 |
|:---:|:---|
| **[1/5]** | **ツール環境の整合性チェック**: `sf-install.sh` を実行し、ツール自体を最新の状態に保ちます。 |
| **[2/5]** | **ブランチの準備**: 現在のGitブランチに対応するリリース用フォルダ (`release/<branch_name>`) を準備します。 |
| **[3/5]** | **Salesforce接続確認**: `sf config get target-org` で接続状態を確認し、未接続ならログインを促します。 |
| **[4/5]** | **VS Code設定の同期**: 接続した組織のエイリアスをプロジェクトのデフォルト設定 (`.sf/config.json` 等) に反映します。 |
| **[5/5]** | **VS Code起動**: `code .` コマンドでプロジェクトをVS Codeで開きます。 |

### `pre-push` フックによる品質保証

1.  開発者が `git push` を実行します。
2.  Gitが自動で `.git/hooks/pre-push` スクリプトを起動します。
3.  `pre-push` スクリプトが `sf-release.sh` を**検証モード (`--dry-run`)** で実行します。
4.  `sf-release.sh` は `release/<branch>/deploy-target.txt` に基づき、変更内容が組織に適用可能かテストします。
5.  検証が成功すれば `push` は続行され、失敗すれば `push` は自動的に中断されます。

## 5. 前提条件

*   Git
*   Salesforce CLI (`sf` コマンド)
*   Visual Studio Code (`code` コマンドがPATHに含まれていること)
*   [推奨] Windows環境の場合は Git Bash

## 6. 設計思想

*   **シンプルさ**: 開発者は `sf-start.sh` を実行するだけでよく、複雑な手順を覚える必要はありません。
*   **透明性**: 実行される処理はステップごとにコンソールに表示され、何が行われているかが明確です。
*   **堅牢性**: 各処理でエラーが発生した場合は、後続の処理を安全に停止し、エラーメッセージを表示します。
*   **環境非依存**: `jq` などの追加ツールに依存せず、Git Bashに含まれる標準的なコマンド (`awk`, `sed` 等) で動作するように作られています。
